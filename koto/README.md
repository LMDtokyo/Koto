# koto/

Главный workspace продукта Koto — наша собственная разработка, чистая
лицензия (НЕ под GPL). Здесь живут все клиентские приложения, которые мы
выпускаем под именем Koto.

## Структура

```
koto/
├── desktop/    Tauri + React + Rust desktop client (Linux/macOS/Windows)
├── android/    (планируется — Compose)
├── ios/        (планируется — Compose Multiplatform)
└── README.md   (этот файл)
```

## Лицензия

Весь код в `koto/` — **наш**, написан с нуля, под нашей лицензией (которая
будет выбрана при первом публичном релизе). Возможные варианты:

- **Closed-source / proprietary** — закрытый код, никаких обязательств перед
  пользователями по публикации исходника. Стандартный путь для коммерческих
  продуктов.
- **MIT / Apache-2.0** — open-source без копилефт-эффекта, можно коммерчески
  использовать без обязательств.
- **AGPL-3.0** — copyleft, требует open-source при сетевом использовании.

Решение принимается ближе к первому публичному релизу.

## Clean-room подход к Session

Мы изучаем структуру и UX Session-Desktop (см. `forks/session/session-desktop/`)
ради **визуальных и архитектурных паттернов**, но:

- ❌ **НЕ копируем** ни одной строки их кода
- ❌ **НЕ переносим** их компоненты, стили, ассеты
- ✅ **Изучаем** иерархию экранов, имена вкладок, расположение кнопок
- ✅ **Переписываем** свой код под понравившиеся паттерны
- ✅ **Документируем** что взяли как идею в [`koto/desktop/SESSION_PATTERNS.md`](desktop/SESSION_PATTERNS.md)

Юридически: GPL покрывает **код**, не **идеи и общую архитектуру**. Clean-room
reimplementation — стандартная практика (так Compaq делал PC BIOS clone в 1980-х,
так Wine делает Win32 API). Ключевое — не смотреть и переписывать буквально, а
**описать паттерн → закрыть Session-вкладку → открыть наш файл → написать с нуля**.
