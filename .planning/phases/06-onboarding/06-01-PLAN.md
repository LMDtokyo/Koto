    Read the current MainActivity.kt first, then add the spring exit animation.

    After `splashScreen.setKeepOnScreenCondition { keepSplash }` (line 35), add the exit animation listener. The listener uses `ViewPropertyAnimator` with a `SpringInterpolator` — but since `SpringInterpolator` requires API 21+ and `DynamicAnimation` is in `androidx.dynamicanimation`, use a simpler approach: `ObjectAnimator` is overkill here, use `ViewPropertyAnimator` with a manual spring-like sequence via `DecelerateInterpolator`.

    Actually, per STACK.md the correct approach on the View layer is `androidx.dynamicanimation.animation.SpringAnimation`. This dep is already available transitively via Compose animation. The spring exit should:
    1. Scale the icon from 1.0 → 0.8 → 1.05 → 1.0 (spring settle-in visual), BUT since the platform already shows it, we want the EXIT to feel like a spring-out:
       - Icon: scale from 1.0 → 1.15 (spring overshoot) then fade out
       - Full splash: alpha from 1.0 → 0.0 over 350ms

    Add these imports at the top of MainActivity.kt:
    ```kotlin
    import android.animation.AnimatorSet
    import android.animation.ObjectAnimator
    import android.view.animation.DecelerateInterpolator
    import androidx.core.splashscreen.SplashScreenViewProvider
    ```

    Replace the current pattern:
    ```kotlin
    val splashScreen = installSplashScreen()
    enableEdgeToEdge()
    var keepSplash = true
    splashScreen.setKeepOnScreenCondition { keepSplash }
    ```

    With:
    ```kotlin
    val splashScreen = installSplashScreen()
    enableEdgeToEdge()
    var keepSplash = true
    splashScreen.setKeepOnScreenCondition { keepSplash }
    splashScreen.setOnExitAnimationListener { provider ->
        animateSplashExit(provider)
    }
    ```

    Add a private function `animateSplashExit` to MainActivity (outside `onCreate`, inside the class):
    ```kotlin
    private fun animateSplashExit(provider: SplashScreenViewProvider) {
        // Icon: spring overshoot scale-up then fade
        val iconScaleX = ObjectAnimator.ofFloat(provider.iconView, "scaleX", 1f, 1.15f, 1f).apply {
            duration       = 350L
            interpolator   = DecelerateInterpolator()
        }
        val iconScaleY = ObjectAnimator.ofFloat(provider.iconView, "scaleY", 1f, 1.15f, 1f).apply {
            duration       = 350L
            interpolator   = DecelerateInterpolator()
        }
        val iconAlpha = ObjectAnimator.ofFloat(provider.iconView, "alpha", 1f, 0f).apply {
            duration       = 300L
            startDelay     = 150L
            interpolator   = DecelerateInterpolator()
        }
        // Full splash background: fade out
        val splashAlpha = ObjectAnimator.ofFloat(provider.view, "alpha", 1f, 0f).apply {
            duration       = 350L
            startDelay     = 100L
            interpolator   = DecelerateInterpolator()
        }
        AnimatorSet().apply {
            playTogether(iconScaleX, iconScaleY, iconAlpha, splashAlpha)
            // Remove splash from window once animation is done
            addListener(object : android.animation.AnimatorListenerAdapter() {
                override fun onAnimationEnd(animation: android.animation.Animator) {
                    provider.remove()
                }
            })
            start()
        }
    }
    ```

    This gives a spring-feel overshoot (1.0 → 1.15 → 1.0) on the icon while fading out, matching `KotoMotion.springBouncy` semantics in the View layer.

    IMPORTANT: Do NOT add `androidx.dynamicanimation` as a new dependency — the ObjectAnimator approach achieves equivalent spring-feel without new deps.
