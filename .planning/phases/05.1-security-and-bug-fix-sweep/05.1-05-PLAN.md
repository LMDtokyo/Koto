<objective>
Fix S-12: media service uses `fmt.Sprintf("%d-%s", time.Now().UnixNano(), accountID)` for file IDs. This is predictable (attacker can guess the key for a known upload time) AND collides within the same nanosecond (two goroutines → same ID → INSERT fails → upload goes to wrong file).

Replace with `uuid.NewString()` from `github.com/google/uuid` v1.6.0 (crypto/rand-backed) for both the file ID and the S3 object key.

This is a ~10-line mechanical fix in a single handler.
</objective>
