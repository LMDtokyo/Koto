<objective>
Close SC-1. "Android client refuses to send a message as plaintext when no session exists; user sees an explicit Encryption unavailable error state rather than a silent plaintext leak".

Root cause: ChatRepository.sendMessage (lines 118-154, with the Base64 fallback at lines 124-133) currently catches any cryptoManager.encrypt failure and Base64-encodes the plaintext as a fallback. The server then stores what looks like ciphertext but is literally Base64(plaintext). Likewise decryptMessage (lines 259-264) Base64-decodes as a fallback, amplifying the leak on both sides.

Fix:
  1. New typed EncryptionUnavailableException (Exception subclass).
  2. Rewrite sendMessage to return Result.failure(EncryptionUnavailableException(...)) when either peerId is missing OR encrypt throws. No Base64 fallback.
  3. Remove the Base64 fallback in decryptMessage — use the placeholder "[зашифровано]".
  4. ChatViewModel: add sendError: String? to ChatState, populate on failure.
  5. ChatScreen: render a simple error banner when sendError is non-null with a dismiss button.
  6. Fill in the Wave 0 unit test with MockK-based assertions.
  7. SC-13: `./gradlew :app:assembleDebug` exits 0 after all edits.

**Important note about the `AndroidBase64` import:** The import `import android.util.Base64 as AndroidBase64` at line 17 of `ChatRepository.kt` is still used by `createDirectConversation` (lines 93-103) to decode Base64-encoded prekey bundle bytes from the server (`bundleDto.identity_key`, signed prekey public, signatures, etc.). This usage is LEGITIMATE — the server returns Base64 strings for binary data in JSON. DO NOT remove the import — only the `AndroidBase64.encodeToString(plaintext...)` fallback in `sendMessage` and the `AndroidBase64.decode(ciphertext...)` fallback in `decryptMessage` must go.

Dependency note: This plan depends on 05.1-07 (OTK persistence). Without OTK persistence, the "retry after processPreKeyBundle" path is fragile on process kill — blocking Base64 BEFORE reliable session restoration would leave users unable to message across restarts. Land 07 first.
</objective>

    suspend fun sendMessage(convId: String, plaintext: String): Result<Unit> {
        val myId = accountPrefs.getAccountId()
            ?: return Result.failure(IllegalStateException("not registered"))

        val peerId = conversationDao.getById(convId)?.peerAccountId?.takeIf { it.isNotEmpty() }
            ?: return Result.failure(
                EncryptionUnavailableException(
                    "No peer identity for conversation $convId — session cannot be established."
                )
            )

        // Encrypt with Signal Protocol (Double Ratchet). SC-1: NO Base64 fallback.
        // If encryption fails, surface EncryptionUnavailableException and do NOT
        // touch chatApi or the local message DB.
        val ciphertext = try {
            cryptoManager.encrypt(peerId, plaintext)
        } catch (t: Throwable) {
            return Result.failure(
                EncryptionUnavailableException(
                    "Signal session with $peerId is not ready. Try again in a moment.",
                    cause = t,
                )
            )
        }

        return runCatching {
            val resp = chatApi.sendMessage(convId, SendMessageRequest(type = 1, ciphertext = ciphertext))

            val expiresAt = if (disappearingMessages.value)
                System.currentTimeMillis() + DISAPPEARING_TTL_MS else 0L
            messageDao.upsert(
                MessageEntity(
                    id             = resp.id,
                    conversationId = convId,
                    ciphertext     = ciphertext,
                    plaintextCache = plaintext,
                    senderId       = myId,
                    myAccountId    = myId,
                    sentAt         = resp.sent_at,
                    delivered      = false,
                    expiresAt      = expiresAt,
                )
            )
            conversationDao.updateLastMessage(convId, plaintext, resp.sent_at)
        }
    }
